Setup

Live in under an hour

No server knowledge needed. No Linux, no firewall rules, no deployment pipelines. Just Godot and your game.

01
Get your Game ID
Contact via YouTube @Champ_gaming. Pick your player count. Pay monthly. You receive your Game ID, server URL, and all 6 GDScript files.
02
Add AutoLoads
Drop the 6 files into your project. Register them as AutoLoads in Project Settings. Set your GAME_ID constant. That's the entire backend setup.
03
Players connect via companion app
Each player installs a small free companion app (Android / Windows). Your game guides them through a one-time setup that takes under a minute.
04
Host a game
One player taps Host. Their session is broadcast to the lobby. Others see it and tap Join. ENet connects directly — low latency, no relay bottleneck.
05
Play — and if host leaves, nobody notices
If the host disconnects or closes the app, the session automatically transitions to the next available player. State is preserved. Game continues.
Your entire multiplayer setup — 4 lines
# Host a session — broadcasts your IP to the lobby func host_game() -> void: NetbirdEnetManager.host_game() # Join a session — IP comes from lobby discovery func join_game(host_ip: String) -> void: NetbirdEnetManager.join_game(host_ip) # Sync your player position every frame StateSync.update_position(global_position) # Save any game stat — persists to cloud StateSync.update_stat("health", current_health)

FAQ

Common questions

GodotConnect uses a lightweight networking layer to give each player a private IP so they can connect directly to each other at low latency. The companion app handles this one-time setup. Players install it once, your game guides them through a 60-second setup, and after that it's automatic every time they play.
The session automatically transitions to the next available player — no one gets kicked, no lobby restart, no lost progress. Player positions and game state are preserved through the transition. This is handled entirely by the included GDScript files, no extra code needed on your side.
No. GodotConnect uses email as a unique identifier only — it is never verified, never emailed to, and never shared. Players can register with any string in email format, real or invented. Password recovery uses a security question, not an email link. This keeps player data private and the system independent of third-party email services.
Yes — GodotConnect uses Godot's native ENet with direct peer-to-peer connections, not a relay server. Latency is as low as the network between players allows, which is far better than WebSocket relay services. It works well for casual to mid-paced competitive games. For sub-20ms tournament-grade shooters you would need dedicated server infrastructure, but for most indie multiplayer games this is the right fit.
