🎮 Example: 3D Player Movement (CharacterBody3D)

Goal: Create a simple player with speed, jump, gravity and a signal.

1. Header

extends CharacterBody3D
class_name Player

2. Declarations

@export var speed: float = 5.0
@export var jump_force: float = 4.5
@onready var camera: Camera3D = $Camera3D
signal died

3. _physics_process(delta)

func _physics_process(delta: float) -> void:
    if not is_on_floor():
        velocity.y -= 9.8 * delta
    
    var direction = Input.get_vector("left", "right", "forward", "back")
    velocity.x = direction.x * speed
    velocity.z = direction.y * speed
    
    if Input.is_action_just_pressed("jump") and is_on_floor():
        velocity.y = jump_force
        died.emit()
    
    move_and_slide()
⚡ Build it in the generator: Add _physics_process function → add if action with condition not is_on_floor() → inside assign velocity.y = velocity.y - 9.8 * delta → then add another if for jump and finally raw action with move_and_slide().

💎 Tips & Best Practices
✅ Extends is mandatory – without it Godot won't know what node your script belongs to. The generator shows a warning if missing.
🧠 Empty type = inferred – leave type empty to let Godot guess the type automatically.
🔌 @onready + $NodePath – perfect for referencing child nodes: @onready var anim: AnimationPlayer = $AnimationPlayer
✏️ Raw action is powerful – when an action doesn't fit the predefined list, use "raw" to write any valid GDScript (e.g. await get_tree().create_timer(1).timeout).
🎯 Signals decouple code – use signal declarations to emit events (damage taken, level up) and connect them in other nodes.
📦 JSON backup – export your script configuration to JSON before closing the browser. Re-import later to continue editing.
