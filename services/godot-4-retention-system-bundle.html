PlayerProfileManager (PPM) is the single bootloader. Nothing runs in _ready() except avatar directory setup. All retention systems wait for a successful login or profile creation before initialising — so there is no risk of any system running before the correct player data is available.

SAVE STRATEGY

Two-tier saves — one cloud call per session

All systems write to a local per-profile file during gameplay. On the next login PPM merges all local files into the profile dictionary and does a single Supabase upsert — then deletes the local files. Cloud saves happen once per session regardless of how active the player is.

save pattern
during gameplay — any system:
PlayerProfileManager.save_retention_local("quests", get_save_data())

on login — PPM only, once:
_merge_local_retention()   ← reads user://profiles/{id}/*.json
await _remote_save(profile_id, current_profile)
_clear_local_retention()   ← deletes local files after sync
PROFILE SWITCH

Mid-game profile changes are fully handled. PPM calls cleanup() on all systems then init() with the new profile. The daily reward UI is freed and re-instanced if today's reward hasn't been claimed yet for the new profile.

Setup

1. Supabase table

One table, one row per player. All retention data lives inside the data JSONB column alongside profile data — no extra tables needed.

SQL
create table public.profiles (
  id         text   not null,
  data       jsonb  not null,
  updated_at timestamp default now(),
  constraint profiles_pkey primary key (id)
);

2. Add autoloads

In Project Settings → Autoloads, add in this order:

Project Settings → Autoloads
PlayerProfileManager   →  PlayerProfile.gd
QuestManager           →  QuestManager.gd
SequenceQuestManager   →  SequenceQuestManager.gd
AchievementManager     →  AchievementManager.gd
CurrencySystem         →  currency_system.gd
SeasonPassManager      →  SeasonPassManager.gd
🚨
Never add DailyReward as an autoload. It extends Control and references node paths in _ready(). Adding it as an autoload crashes the project on startup before any scene loads.

3. Wire LevelSystem and DailyReward

LevelSystem lives inside your ProfileManagerUI scene. Its UI script calls PlayerProfileManager.register_level_system(level_system) in _ready(). In your Main scene assign the DailyReward packed scene to the export var — PPM instances and frees it automatically.

Main.gd
@export var daily_reward_scene: PackedScene

func _ready() -> void:
    PlayerProfileManager.retention_ready.connect(_on_retention_ready)
    PlayerProfileManager.logout_occurred.connect(_free_daily_reward)

Adding new systems

Every new retention system follows the exact same three-function pattern. Create your script, add it as an autoload, and register it in PPM:

any new system
func init(username: String, profile: Dictionary) -> void:
    # read your slice from profile, connect signals
    var saved = profile.get("my_system", {})

func get_save_data() -> Dictionary:
    return your_state.duplicate(true)

func cleanup() -> void:
    # disconnect signals, clear state

# save from anywhere inside that system:
PlayerProfileManager.save_retention_local("my_system", get_save_data())
