If You Lose Your Keystore Password

NO RECOVERY POSSIBLE!

If you've lost the password to a keystore you created yourself:

  • There's no way to recover it
  • You'll need to create a new keystore
  • You cannot update existing published apps
  • Must publish as new app (lose all users/reviews)

‚úÖ What You Should Do

For normal exports that you want to publish or distribute:

  1. Create your own secure release keystore
  2. Save credentials in a password manager
  3. Backup keystore file in multiple locations
  4. Configure in Godot under Project ‚Üí Export ‚Üí Android

üîë Create Release Keystore with Keytool

Complete Keytool Command Guide

keytool -genkey -v -keystore my-release.keystore -alias my_app -keyalg RSA -keysize 2048 -validity 10000

üí° What Each Part Means:

  • -genkey: Generate a new key
  • -v: Verbose output (shows details)
  • -keystore my-release.keystore: Output filename
  • -alias my_app: Your app's alias/username
  • -keyalg RSA: Encryption algorithm
  • -keysize 2048: Key size (secure)
  • -validity 10000: Valid for ~27 years

What You Need to Remember & Save

‚ö†Ô∏è SAVE THIS INFORMATION IMMEDIATELY!

From this process, you must save:

CRITICAL INFORMATION TO SAVE:
1. Keystore Password - The password you type at the very first prompt 2. Alias - In the command, it's "my_app" (you can change this) 3. Key Password - Usually same as keystore password 4. Keystore File - "my-release.keystore" (save the file!) 5. Personal Details - Name, organization, country for future reference

üìù Where to Save:

  • Password Manager: Bitwarden, LastPass, 1Password
  • Cloud Storage: Google Drive, Dropbox (encrypted)
  • Physical Backup: USB drive, external hard drive
  • Printed Copy: Keep in a secure location

Configure Keystore in Godot

In Godot Editor ‚Üí Project ‚Üí Export ‚Üí Android:
// Use the information from your keytool command: Release Keystore: [Browse to "my-release.keystore" file] Release User: "my_app" (the alias from command) Release Password: [The keystore password you entered] // Example configuration: Keystore/Release: C:\GameDev\Keystores\my-release.keystore Keystore/Release User: my_app Keystore/Release Password: YourStrongPassword123! // REQUIRED Permissions: [x] INTERNET [x] ACCESS_NETWORK_STATE [x] VIBRATE (if your game uses vibration)

üí° Important Notes:

  • Release User = The alias from keytool command
  • Release Password = The keystore password (NOT alias password)
  • Keystore file = The .keystore file you created
  • These settings are for Release builds only
