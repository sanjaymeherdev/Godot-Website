System Architecture

How It Works

┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│   Website   │───▶│   Edge      │───▶│  Razorpay/  │
│  (Products) │    │  Function   │    │   PayPal    │
└─────────────┘    └─────────────┘    └─────────────┘
                          │                    │
                          ▼                    ▼
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│   Godot     │◀───│  Redeem     │◀───│  Email +    │
│   Game      │    │  Codes DB   │    │  Redeem Code│
└─────────────┘    └─────────────┘    └─────────────┘

User buys → Payment processed → Redeem code generated → Email sent → Godot validates code → Product unlocked

Database Setup

Create the `redeem_codes` table

Supabase SQL Editor 
CREATE TABLE public.redeem_codes (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  code varchar(6) UNIQUE,
  email text NOT NULL,
  product_id text NOT NULL,
  product_name text NOT NULL,
  payment_provider text CHECK (payment_provider IN ('razorpay', 'paypal')),
  payment_provider_id text,
  amount numeric NOT NULL,
  currency text NOT NULL CHECK (currency IN ('INR', 'USD')),
  payment_status text DEFAULT 'pending',
  used boolean DEFAULT false,
  used_at timestamptz,
  created_at timestamptz DEFAULT now(),
  updated_at timestamptz DEFAULT now()
);

CREATE INDEX idx_redeem_codes_code ON redeem_codes(code);
CREATE INDEX idx_redeem_codes_email ON redeem_codes(email);
⚠️
Your existing `products` table should have: `uuid`, `itemid`, `itemname`, `price`, `type`, `category`, `texture`

Edge Function (gamestore-redeem)

Deploy to Supabase

Terminal 
# Deploy the edge function
supabase functions deploy gamestore-redeem

# Set required secrets
supabase secrets set RP_LIVE_API=your_razorpay_key_id
supabase secrets set RP_LIVE_SECRET=your_razorpay_secret
supabase secrets set PP_LIVE_API=your_paypal_client_id
supabase secrets set PP_LIVE_SECRET=your_paypal_secret
supabase secrets set RESEND_API_KEY=your_resend_key
✅
The edge function handles 3 actions: createPayment (Razorpay/PayPal), pollStatus (check payment), redeem (Godot validation)

Godot Integration

Redeem Code Script

redeem.gd 
extends Node

const EDGE_URL = "https://your-project.supabase.co/functions/v1/gamestore-redeem"

func redeem_product(email: String, code: String, product_id: String):
    var http = HTTPRequest.new()
    add_child(http)
    
    var body = JSON.stringify({
        "action": "redeem",
        "email": email,
        "code": code,
        "product_id": product_id
    })
    
    var headers = ["Content-Type: application/json"]
    http.request(EDGE_URL, headers, HTTPClient.METHOD_POST, body)
    http.request_completed.connect(_on_redeem_completed)

func _on_redeem_completed(result, response_code, headers, body):
    var json = JSON.new()
    json.parse(body.get_string_from_utf8())
    var response = json.data
    
    if response.success:
        print("✅ Unlocked: ", response.product_id)
    else:
        print("❌ Failed: ", response.message)

Payment Providers

Razorpay (INR) & PayPal (USD) Setup

⚠️
Razorpay: Enable "Payment Links" feature in dashboard. Create API keys (Live mode for production).
⚠️
PayPal: Create a REST API app in Developer Dashboard. Get Client ID and Secret. Enable webhooks with your edge function URL.
✅
Resend (Email): Create API key. Verify your domain. Add `RESEND_API_KEY` and `RESEND_FROM_EMAIL` to secrets.

Testing Guide

Test Flows

  • Test with [TEST] mode on Razorpay/PayPal first
  • Use test card numbers (Razorpay: 4111 1111 1111 1111, PayPal: buyer@personal.com)
  • Verify redeem code appears in success popup
  • Check email for delivery (Resend logs)
  • Test Godot redemption with the received code
  • Verify `redeem_codes` table shows `used: true` after redemption
🚨
Before production: Swap test API keys for LIVE keys. Update all webhook URLs. Test with real small payments (₹10 / $1).
