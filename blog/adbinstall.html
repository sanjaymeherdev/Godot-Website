How to Install a Debug APK Using ADB (Test Mode)

For business owners & founders who need to QA their app before launch

So your developer sent you a .apk file and said "install it via ADB in test mode."

If you're not a developer, that sounds like jargon. But here's the truth: installing a debug APK is simpler than setting up a new WhatsApp Business account. Let me show you how.

Why ADB? Why not just tap the file?

A debug APK is a test version of your app. It's not signed for production, which means your phone won't install it directly by tapping the file – as a security feature. ADB (Android Debug Bridge) is the official, safe way to install test builds.

Good news: once you set this up once, every future test install takes less than 10 seconds.

What You'll Need

  • Windows, Mac, or Linux computer
  • Your Android phone + USB cable
  • The debug APK file (your developer gave you this)
  • 5–10 minutes
1 Enable Developer Options on Your Phone
ActionWhat to do
Go to Settings → About PhoneScroll to Build Number
Tap Build Number seven times fastYou'll see: "You are now a developer!"
Go back to Settings → System → Developer OptionsIt should now appear near the bottom
⚠️ No toast? Some phones (Xiaomi, OnePlus, Samsung) require you to enter your lock screen PIN after a few taps. Just keep tapping.
2 Turn On USB Debugging

Inside Developer Options, find USB Debugging and toggle it ON.

Your phone will show a warning – tap OK.

3 Install ADB on Your Computer (One-Time Setup)

Windows

  • Download Platform Tools from Google: developer.android.com/tools/releases/platform-tools (Download the Windows .zip file)
  • Extract the folder to C:\platform-tools
  • Open Command Prompt, type: cd C:\platform-tools

Mac

  • Open Terminal and run: /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)" (This installs Homebrew)
  • Then run: brew install android-platform-tools

Linux (Ubuntu/Debian)

  • In Terminal: sudo apt update && sudo apt install adb
4 Connect Your Phone & Allow Debugging
  1. Plug your phone into your computer via USB
  2. On your phone, you'll see: "Allow USB debugging?" Check "Always allow from this computer" and tap OK
  3. On your computer, verify the connection: adb devices
List of devices attached
XXXXXXXXX    device
If you see unauthorized instead of device, just unplug, replug, and tap Allow on the popup again.
5 Install the Debug APK (with -t flag)

Copy your APK file into the ADB folder:

  • Windows: C:\platform-tools\your-app.apk
  • Mac/Linux: wherever your Terminal is open

Run this command (notice the -t flag — this is required for debug/test builds):

adb install -t your-app.apk
🔑 Why -t? Android blocks debug APKs by default. The -t flag tells ADB: "Yes, I know this is a test build. Install it anyway."

Success looks like:

Performing Streamed Install
Success
💡 Pro tip: If you forget the -t, you'll see INSTALL_FAILED_TEST_ONLY. Just add -t and run again.

For future installs (same app, updated version)

If you're updating the same debug app:

adb install -t -r your-new-app.apk

The -r flag reinstalls/updates the existing app without losing your test data.

Troubleshooting (Updated)

ProblemFix
INSTALL_FAILED_TEST_ONLYYou forgot -t. Run: adb install -t your-app.apk
adb: command not foundYou're not in the platform-tools folder. cd into it first.
device unauthorizedUnplug USB, replug, and tap Allow on the phone screen.
INSTALL_FAILED_UPDATE_INCOMPATIBLEUninstall old version first: adb uninstall com.your.app.package
Phone not showing upTry a different USB cable. Many cheap cables are charge-only.

Every Future Install (After Day One)

  • Plug phone into computer
  • Run adb install -t your-new-app.apk
  • Done. ~10 seconds.
