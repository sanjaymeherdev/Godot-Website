You're Using DEBUG Keystore!

Godot auto-creates a debug keystore. Here's why it won't work for stores.

Understand the Difference ‚Üí

üö® IMPORTANT: You CANNOT Use Debug APK for Stores!

The "signed" APK you're seeing is signed with DEBUG keystore.

Stores like Amazon and AdMob require RELEASE keystore.

DEBUG vs RELEASE Keystore

Feature DEBUG Keystore (What you have) RELEASE Keystore (What you need)
Who creates it? Godot automatically YOU create it manually

For Amazon Store? ‚ùå NO - Rejected ‚úÖ YES - Accepted

‚ö†Ô∏è Problems with Debug Keystore:

  • Different for each computer - Can't share with team
  • Expires in 1 year - APK stops working
  • Password is "android" - Not secure
  • Changes when you reinstall Godot - Can't update app

üîç Check What Type of APK You Have

Method 1: Check in Godot

Go to Project ‚Üí Export ‚Üí Android

Look at Keystore/Release field:

  • Empty = Using debug keystore
  • Has path = Using release keystore

Method 2: Command Line

keytool -list -v -keystore your_app.apk

If it shows "CN=Android Debug" = DEBUG

If it shows YOUR name = RELEASE

Method 3: Try Uploading

Try uploading to Amazon App Store:

  • DEBUG APK: Will be rejected
  • RELEASE APK: Will be accepted

Quick Test Script for Windows:

@echo off
echo Checking APK signature...
keytool -list -printcert -jarfile "your_app.apk"
pause

Save as check_apk.bat and run

1

Step 1: Generate Release Keystore in Godot

1. Project ‚Üí Export...
2. Select Android preset
3. Scroll to Keystore/Release
4. Click "..." button
5. Select "New Keystore..."
6. Fill details (25 years!)
7. Save to SAFE location
8. Remember password!
Save to: D:\GameDev\Keystores\mygame_release.keystore
2

Step 2: Export RELEASE APK (Not Debug)

CRITICAL: When exporting:

‚úÖ DO:
1. Select Android preset
2. Click "Export Project"
3. Choose "Release" mode
4. Save as: mygame_release.apk

‚ùå DON'T:
- Don't use "One-click deploy"
- Don't export as Debug
- Don't use "Export & Run"
3

Step 3: Verify Your New APK

Check if it's Release:

keytool -list -printcert -jarfile "mygame_release.apk"

Expected output:
Owner: CN=Your Name, OU=Development, O=Your Company...
NOT: CN=Android Debug
