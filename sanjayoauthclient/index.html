🎯 What is SanjayOauthClient?

🔐 Simple OAuth Client

A lightweight authentication layer that lets users sign in with their Google account. The app does not store passwords, access emails for marketing, or read Drive/Gmail/Calendar data. It only confirms identity and provides a session token.

✅ Perfect for dashboards, internal tools, or any app that needs "Login with Google" without extra permissions.

⚡ Why this app exists

Many apps request excessive access. SanjayOauthClient follows least-privilege principle — we ask only for what we need: your public profile and email address. Nothing more.

OpenID Connect OAuth 2.0 Google Identity Services
🔍 Requested information (scopes)

📌 What we access

  • ✅ openid – your Google identifier (sub)
  • ✅ email – your email address and verification status
  • ✅ profile – your name and profile picture (optional avatar)
🔹 https://www.googleapis.com/auth/userinfo.email 🔹 https://www.googleapis.com/auth/userinfo.profile 🔹 openid

🚫 What we NEVER request

  • ❌ Google Drive files
  • ❌ Gmail / Contacts / Calendar
  • ❌ YouTube or Analytics data
  • ❌ Location, phone number, or device sensors

📌 No sensitive scopes – only "non-sensitive" OAuth scopes as per Google policy.

⚙️ How the login flow works

1️⃣ User clicks "Sign in with Google"

Redirects to Google’s OAuth consent screen. The user sees exactly which data is requested (email + profile).

2️⃣ User approves access

Google returns an ID token to SanjayOauthClient. The token is verified on our server (or client-side for demo).

3️⃣ Authenticated session starts

The user’s name, email, and avatar are displayed inside the protected area. No background sync, no data mining.

4️⃣ Logout & session end

User can sign out anytime. No long-term storage of personal data unless user chooses to save preferences.
