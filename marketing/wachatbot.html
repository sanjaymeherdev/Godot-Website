How one rule works

Trigger → reply → branch → follow-up

Every rule you build is this same chain. Set it up once in the visual builder; it runs on every conversation after that.

TRIGGER

Keyword match

"book", "schedule", "meeting" — exact, contains, or fuzzy matching on what the customer types.

ACTION

Template or AI reply

Send a rich template — buttons, list, product card — or let an AI prompt answer using your docs.

CONDITION

Branch on the reply

If they pick "10:00 AM", check a sheet, an order status, or cart value — and reply differently.

FOLLOW-UP

Nudge, automatically

No reply after 4 hours? No purchase? Send a follow-up template without lifting a finger.

rule.json — generated by the visual builder 
{
  "rule": "Book a meeting",
  "keywords": ["book", "schedule", "meeting"],
  "match_type": "contains",
  "action": { "type": "template", "template_id": "meeting_slots" },
  "conditions": [
    { "variable": "reply_option", "operator": "equals",
      "value": "10:00 AM", "template_id": "store_hours" }
  ],
  "follow_up": { "enabled": true, "hours": 4,
    "condition": "no_reply", "template_id": "store_hours" }
}
