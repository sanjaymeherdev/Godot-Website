Supabase Setup – SQL + RLS

Database Schema

supabase_schema.sql 
CREATE TABLE profiles (
    id TEXT PRIMARY KEY,
    data JSONB NOT NULL,
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_profiles_id ON profiles(id);

Row Level Security (RLS)

rls_policies.sql 
ALTER TABLE profiles ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Users can read own profile" ON profiles
    FOR SELECT USING (auth.uid() = id);

CREATE POLICY "Users can update own profile" ON profiles
    FOR UPDATE USING (auth.uid() = id);

CREATE POLICY "Users can insert own profile" ON profiles
    FOR INSERT WITH CHECK (auth.uid() = id);
🔐
Environment variables needed: Set SUPABASE_URL and SUPABASE_ANON in your Godot project or PlayerProfileManager.gd constants.

Cloud + Local Save – Already Integrated

How It Works

  • All systems save to user://profiles/[profile_id]/ immediately on any change
  • On save_profile(), local JSON files merge into cloud (Supabase)
  • On login, cloud data loads and overrides local (if newer)
  • Offline-first – game works without internet, syncs when back online
    • 🔓
    Anti-cheat options: Local saves can be manipulated by advanced users. For competitive games, you can:

    → Enable direct cloud save only (disable local fallback)
    → Implement local save encryption (built-in option available)
    → Use server-side validation for critical currencies/progress

    💾
    Zero setup required: PlayerProfileManager automatically handles save/load. Each system implements get_save_data() and init() – you never write save/load code.
