HOW A PROMPT BECOMES A SCENE

It doesn't guess. It plans first, then builds file by file.

Most "AI game maker" demos are one giant prompt that either works or doesn't. This is a real pipeline — a manifest gets planned, then each file gets generated and checked off in order, so you always know exactly what's been built and what's still pending.

01

You describe the game

plain language, not a spec

"A top-down twin-stick shooter with waves of enemies" is enough. If anything's ambiguous, the agent asks — it doesn't silently guess and hand you the wrong thing.

02

It plans the manifest, not the code

JSON file list — scripts, builders, outputs

Logic scripts are listed before the builder scripts that depend on them. 2D requests get 2D node rules (CharacterBody2D, Area2D). 3D requests get the player/main split kept reusable. The plan is visible before a single line of code is written.

03

Each file gets generated and written in

strict scene-tree conventions, no >= drift

Builder scripts follow a fixed hierarchy and positioning math — not whatever the model felt like that run. Output goes straight into your project structure, not a chat window you copy from.

04

Scenes get packed, saved, and scanned

ResourceSaver → .tscn → filesystem refresh

Builders run as real EditorScripts inside Godot — they pack the scene, save it to res://scenes/, and trigger a filesystem scan so it shows up in your editor immediately.

05

Stop and resume whenever

manifest.json persists state

Close the editor mid-build, come back tomorrow — the checklist remembers exactly which files were done and which are still pending. Nothing has to start over.

BEFORE YOU ASK

Quick answers.

Do I need to know GDScript to use this?
No — but the more specific your prompt, the better the output. It writes real, readable GDScript, so it's also a fast way to learn the language by example.
Does it work for both 2D and 3D projects?
Yes — the planning rules are genuinely different for each, not a generic template stretched to fit both. 3D always keeps the player scene separate and reusable; 2D adapts based on whether you want one scene or several.
What if the prompt is too vague?
It asks a clarifying question instead of guessing — you'll see an "ASK_USER" prompt in the panel rather than a wrong build silently appearing in your project.
Can I stop mid-build and come back later?
Yes. The manifest is saved to disk as you go — close Godot, reopen tomorrow, and it picks up exactly where it left off.
