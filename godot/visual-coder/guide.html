5. Advanced: Match Statement Builder

The match action replicates Godot's powerful pattern matching. Add patterns, values and nested actions.

match weapon_type:
    WeaponType.SWORD:
        damage = 25
    WeaponType.BOW:
        damage = 15
    _:
        damage = 5
➕ Pattern tips:
• Use _ (underscore) as default/wildcard pattern.
• Separate multiple values with commas: 1, 2, 3
• Each pattern can have multiple actions (indented automatically).

🎮 Live Example — 3D Player Controller

🔧 Configuration

extends CharacterBody3D
class_name Player

@export var speed: float = 5.0
@export var jump_force: float = 4.5
@onready var camera: Camera3D = $Camera3D
signal player_jumped

⚙️ _physics_process

func _physics_process(delta: float) -> void:
    # gravity
    if not is_on_floor():
        velocity.y -= 9.8 * delta
    
    # input movement
    var direction = Input.get_vector("left", "right", "forward", "back")
    velocity.x = direction.x * speed
    velocity.z = direction.y * speed
    
    # jump + signal
    if Input.is_action_just_pressed("jump") and is_on_floor():
        velocity.y = jump_force
        player_jumped.emit()
    
    move_and_slide()
🛠️ Build it in the generator: Add _physics_process(delta) → add if not is_on_floor() → inside, assign velocity.y → add jump if with Input.is_action_just_pressed → raw action move_and_slide().

💎 Best Practices & Hardcoded Rules
✅ Always set extends — no extends = no attachment to nodes.
🧠 Type inference vs explicit — use := when variable is initialized on the same line.
🔌 @onready + $ syntax — optimal pattern for child nodes: @onready var anim := $AnimationPlayer
✏️ Raw action flexibility — write await get_tree().create_timer(1.0).timeout, queue_free(), any expression.
🎯 Signals are first-class — decouple systems; emit signals on death, hit, level up.
📦 JSON backup workflow — export before closing browser, re-import to continue editing complex scripts.
